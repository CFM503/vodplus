问题描述:
---------
Cloudflare Pages 构建失败，错误信息:
- "stream did not contain valid UTF-8"
- "Unexpected character '\0' in src/config/config.ts"

根本原因:
---------
1. src/config/config.ts 文件在 JSON 对象关闭后(第102行)被注入了
   无效的 null 字节 (\\x00) 和额外的 PREFETCH_CONFIG 代码段
2. src/app/page.tsx 文件使用了 UTF-16-LE 编码而不是 UTF-8

修复措施:
---------
? 清理 src/config/config.ts - 移除 null 字节和额外代码 (113行 -> 103行)
? 转换 src/app/page.tsx - 从 UTF-16-LE 转为 UTF-8，移除 BOM
? 更新 package.json 版本: 0.6.0 -> 0.6.1

文件变更:
---------
修改的文件:
  - src/config/config.ts    (3980 -> 3557 bytes)
  - src/app/page.tsx        (7898 -> 3951 bytes)
  - package.json            (版本更新)

如何使用:
---------
方法 1: 运行 push.bat (推荐)
  双击 push.bat 文件，它将自动:
  - 添加所有修改文件到 git
  - 创建带有描述信息的提交
  - 推送到远程仓库

方法 2: 手动推送
  在项目目录中运行以下命令:
  
  git add package.json src/config/config.ts src/app/page.tsx
  git commit -m "fix: 解决 Cloudflare Pages 部署编码问题"
  git push origin main

验证:
-----
所有源文件现在都是有效的 UTF-8 编码:
  ? 无 null 字节
  ? 无 BOM 字符
  ? 正确的行结束符
